# ドキュメント最新化

以下の手順でドキュメントを最新化してください。

## 手順

### 1. 前回の同期コミットを確認

`docs/SYNC.md` を読み、前回同期時のコミットハッシュを取得する。

### 2. 差分コミットを確認

```bash
git log <前回ハッシュ>..HEAD --oneline
```

差分がなければ「ドキュメントは最新です」と報告して終了する。

### 3. 変更内容を調査

差分コミットに含まれる変更ファイルを確認し、ドキュメントへの影響を分析する。

```bash
git show --stat <コミットハッシュ>
```

影響を受ける可能性のあるドキュメント:
- `docs/usage/` — 各画面の使い方
- `docs/setup/` — セットアップ手順
- `docs/README.md` — 目次・概要

### 4. ドキュメントを更新

実装と乖離している箇所を修正する。UIの見た目のみの変更（CSSクラス変更など）は原則ドキュメント更新不要。機能追加・削除・仕様変更は必ず反映する。

### 5. `docs/SYNC.md` を更新

最新のコミットハッシュと日付・コミットメッセージに書き換える。

```markdown
# ドキュメント同期状態

最後にドキュメントを実装と照合したコミット:

\`\`\`
<新しいハッシュ>  <日付>  <コミットメッセージ>
\`\`\`

次回 docs 更新時は以下のコマンドで差分を確認する:

\`\`\`bash
git log <新しいハッシュ>..HEAD --oneline
\`\`\`
```

### 6. コミット・プッシュ

```bash
git add docs/
git commit -m "docs: ドキュメントを最新の実装に同期"
git push
```
