# 実績入力（ワークログ）

**URL**: `/work-logs`

週次タイムライン形式で各メンバーの実績（作業時間）を入力・管理する画面です。

---

## 画面構成

Outlook 風の週表示タイムラインカレンダーで実績を管理します。

- **縦軸**: 時間軸（5:00 〜 22:00）
- **横軸**: 曜日（月〜日）
- **土日・祝日**: 背景色でハイライト表示
- **祝日名**: 各日付ヘッダーに祝日名を表示

### 初期表示範囲

実績入力の定時開始・終了（設定 → 一般設定で変更可）を中心にスクロール位置が設定されます。

| 設定項目 | デフォルト |
|---|---|
| 定時開始 | 10:00 |
| 定時終了 | 19:00 |

---

## 週の切り替え

画面上部の「<」「>」ボタンで前後の週に移動できます。現在表示中の週の期間（`YYYY/MM/DD 〜 YYYY/MM/DD`）が中央に表示されます。

---

## メンバーフィルタ

画面上部のドロップダウンでメンバーを切り替えると、選択したメンバーの実績のみ表示されます。

デフォルトはログインユーザー自身の実績が表示されます。

---

## 実績の入力

### 新規登録

カレンダーのグリッド上をドラッグして時間枠を選択すると、入力モーダルが開きます。

| 入力項目 | 必須 | 説明 |
|---|---|---|
| メンバー | ✅ | 実績の対象メンバー |
| 実績種別 | ✅ | カテゴリグループから選択 |
| 開始時刻 | ✅ | 実績の開始時刻 |
| 終了時刻 | ✅ | 実績の終了時刻 |
| エピック | - | 関連するエピック（案件） |
| イシュー | - | 関連する GitHub Issue |
| 備考 | - | メモ・補足 |

工数は開始・終了時刻から自動計算されます。

### 編集・削除

登録済みのブロックをクリックするとモーダルが開き、編集・削除ができます。

ドラッグで移動、下端をドラッグでリサイズも可能です。

---

## 工数集計

実績種別のグループに「工数あり」フラグが設定されているカテゴリの実績のみ集計対象になります（`/settings/work-log-categories` で設定）。

「工数なし」グループ（会議・社内イベントなど）の実績は記録されますが、集計レポートには含まれません。
