# レトロスペクティブ

**URL**: `/retrospectives`

スプリントごとの振り返り（KPT）を記録・管理する画面です。

---

## スプリント選択

画面右上のドロップダウンでスプリントを選択します。進行中のスプリントには `(進行中)` のラベルが付きます。

スプリントを切り替えると、そのスプリントの KPT 一覧が表示されます（URL の `?sprint_id=` パラメータで管理）。

**初期スプリント選択ロジック（URL パラメータなしの場合）:**

1. 今日の日付が期間内に含まれるスプリント
2. `open` 状態のスプリント
3. 最新のスプリント（`end_date` 降順）

---

## タブ構成

画面は「KPT入力」と「履歴」の2タブ構成です。

### KPT入力タブ

選択中のスプリントの KPT を入力・確認するタブです。

フォーム上部にスプリントの期間（`YYYY-MM-DD 〜 YYYY-MM-DD`）が表示されます。

### 履歴タブ

全スプリントの KPT をまとめて確認するタブです。

- 列 = スプリント（左が新しい順、右が古い順）
- 行 = Keep / Problem / Try の3区分
- データが1件以上あるスプリントのみ表示されます
- 横スクロールで過去のスプリントを閲覧できます

---

## KPT の追加

スプリントを選択すると入力フォームが表示されます。

| 入力項目 | 説明 |
|---|---|
| 種別（ラジオ） | `Keep` / `Problem` / `Try` から選択 |
| 内容 | テキストエリアに内容を入力（必須） |

「追加」ボタンで登録します。

---

## KPT ボード

登録済みの KPT は種別ごとに3列のカードボードで表示されます。

| 種別 | 色 | 説明 |
|---|---|---|
| Keep | 緑 | うまくいっていること・継続したいこと |
| Problem | 赤 | 問題・課題・改善が必要なこと |
| Try | 青 | 次のスプリントで試してみること |

---

## KPT の編集・削除

各カードの「編集」ボタンからインライン編集ができます。

- **編集**: テキストエリアが展開され、「保存」で確定、「キャンセル」で元に戻ります
- **削除**: 確認ダイアログが表示され、確認後に削除されます

---

## よくある質問

**Q: スプリントが選択できない**
→ リポジトリに `github_project_number` が設定されている場合は「GitHub 同期」ボタンを押してください。GitHub Projects の Sprint Iteration からスプリントが作成されます。設定されていない場合はスプリントの作成が必要です（詳細は [スプリント](./sprints.md) を参照）。

**Q: 過去のスプリントの KPT をまとめて見たい**
→ 「履歴」タブをクリックすると、全スプリントの KPT を横並びで比較できます。
