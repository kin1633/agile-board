# 設定

サイドバーの「設定」をクリックするとアコーディオン展開し、以下のサブページに直接移動できます。

| メニュー | URL | 概要 |
|---|---|---|
| 一般 | `/settings/general` | 工数計算の基準時間など全体設定 |
| リポジトリ | `/settings/repositories` | GitHub リポジトリの追加・同期設定 |
| メンバー | `/settings/members` | チームメンバーの表示名・稼働時間 |
| ラベル | `/settings/labels` | ベロシティ計算からの除外設定 |

---

## 一般設定

**URL**: `/settings/general`

工数計算に関するアプリ全体の設定を管理します。

| 設定項目 | 説明 | デフォルト |
|---|---|---|
| 1人日の基準時間 | 工数を人日に換算する際の基準（例: 7h → 7h ÷ 7h/人日 = 1人日） | 7時間 |

「保存」ボタンをクリックすると設定が即座に保存されます。

---

## リポジトリ管理

**URL**: `/settings/repositories`

GitHub から同期するリポジトリを管理します。

### リポジトリの追加

「+ リポジトリを追加」ボタンをクリックするとフォームが展開します。

| 入力項目 | 必須 | 説明 |
|---|---|---|
| オーナー | ✅ | GitHub のオーナー名（例: `octocat`） |
| リポジトリ名 | ✅ | リポジトリ名（例: `hello-world`） |

追加後は `{owner}/{name}` の形式で一覧に表示されます。

### 有効・無効の切り替え

各リポジトリ行の「有効にする」/「無効にする」ボタンで `active` フラグを切り替えます。

- **有効（active）**: GitHub 同期の対象になります
- **無効**: 同期がスキップされます

### GitHub Project Number の設定

各リポジトリ行の `Project #` 列に GitHub Projects v2 のプロジェクト番号を入力すると、**Iteration モード**が有効になります。

- **設定あり**: GitHub Projects の Iteration がスプリントとして同期されます
- **設定なし（デフォルト）**: GitHub マイルストーンがスプリントとして同期されます（後方互換）

プロジェクト番号は GitHub Projects の URL から確認できます（例: `https://github.com/orgs/myorg/projects/5` → `5`）。

> Iteration モードを初めて使用する場合は、一度ログアウト → 再ログインして `project` スコープを再取得してください。

### 最終同期日時

リポジトリ名の下に最終同期日時が表示されます。未同期の場合は「未同期」と表示されます。

## メンバー管理

**URL**: `/settings/members`

チームメンバーの情報を管理します。

### メンバーの自動登録

メンバーは **GitHub OAuth でログインした時点で自動的に登録**されます。手動追加・削除はできません。

### メンバーの編集

各メンバー行の「編集」ボタンから以下の項目を変更できます。

| 項目 | 説明 |
|---|---|
| 表示名 | アプリ内での表示名（デフォルトは GitHub ログイン名） |
| 1日の稼働時間 | 担当者別工数グラフ等で使用（デフォルト: 6時間/日） |

GitHub ログイン名（`@username`）は変更できません。

---

## ラベル管理

**URL**: `/settings/labels`

GitHub Issue のラベルに対して、ベロシティ計算の除外設定を行います。

### ベロシティ除外フラグ

各ラベルのトグルスイッチで `exclude_velocity` を切り替えます。

- **オン（除外）**: このラベルが付いた Issue はベロシティ計算から除外されます
- **オフ（含める）**: 通常通りベロシティに含まれます

> ラベル一覧は GitHub 同期後に表示されます。同期前は空欄になります。

### 用途例

`spike`・`tech-debt`・`管理` などの非開発 Issue を除外することで、実際の開発速度をより正確に計測できます。

---

## プロフィール設定

**URL**: `/settings/profile`

ログインユーザー自身の名前とメールアドレスを変更できます。

---

## セキュリティ設定

**URL**: `/settings/security`

パスワードの変更ができます。

> このアプリは GitHub OAuth 認証を使用しているため、パスワードは補助的なものです。2要素認証（2FA）はサポートしていません。
