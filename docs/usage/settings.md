# 設定

サイドバーの「設定」をクリックするとアコーディオン展開し、以下のサブページに直接移動できます。

| メニュー | URL | 概要 |
|---|---|---|
| 一般 | `/settings/general` | 工数計算の基準時間・外観・実績入力設定 |
| リポジトリ | `/settings/repositories` | GitHub リポジトリの追加・同期設定 |
| メンバー | `/settings/members` | チームメンバーの表示名・稼働時間 |
| ラベル | `/settings/labels` | ベロシティ計算の管理対象ラベル設定 |
| 実績種別 | `/settings/work-log-categories` | 実績入力の種別（カテゴリ）・グループを管理 |
| 祝日 | `/settings/holidays` | 日本の祝日・特別休日を管理 |

---

## 一般設定

**URL**: `/settings/general`

工数計算に関するアプリ全体の設定と外観を管理します。

| 設定項目 | 説明 | デフォルト |
|---|---|---|
| 外観 | ライト・ダーク・システム連動を切り替え（クッキーに保存） | システム |
| 1人日の基準時間 | 工数を人日に換算する際の基準（例: 7h → 7h ÷ 7h/人日 = 1人日） | 7時間 |
| 実績入力の定時開始 | 実績入力カレンダーのデフォルト表示開始時刻 | 10:00 |
| 実績入力の定時終了 | 実績入力カレンダーのデフォルト表示終了時刻 | 19:00 |

外観の切り替えはボタン選択後すぐに反映されます。その他の設定は「保存」ボタンをクリックすると保存されます。

## リポジトリ管理

**URL**: `/settings/repositories`

GitHub から同期するリポジトリを管理します。

### リポジトリの追加

「+ リポジトリを追加」ボタンをクリックするとフォームが展開します。

| 入力項目 | 必須 | 説明 |
|---|---|---|
| オーナー | ✅ | GitHub のオーナー名（例: `octocat`） |
| リポジトリ名 | ✅ | リポジトリ名（例: `hello-world`） |

追加後は `{owner}/{name}` の形式で一覧に表示されます。

### 有効・無効の切り替え

各リポジトリ行の「有効にする」/「無効にする」ボタンで `active` フラグを切り替えます。

- **有効（active）**: GitHub 同期の対象になります
- **無効**: 同期がスキップされます

### GitHub Project Number の設定

各リポジトリ行の `Project #` 列に GitHub Projects v2 のプロジェクト番号を入力すると、**Iteration モード**が有効になります。

- **設定あり**: GitHub Projects の Iteration がスプリントとして同期されます
- **設定なし（デフォルト）**: スプリントの同期は行われません

プロジェクト番号は GitHub Projects の URL から確認できます（例: `https://github.com/orgs/myorg/projects/5` → `5`）。

> Iteration モードを初めて使用する場合は、一度ログアウト → 再ログインして `project` スコープを再取得してください。

### 最終同期日時

リポジトリ名の下に最終同期日時が表示されます。未同期の場合は「未同期」と表示されます。

## メンバー管理

**URL**: `/settings/members`

チームメンバーの情報を管理します。

### メンバーの自動登録

メンバーは **GitHub OAuth でログインした時点で自動的に登録**されます。手動追加・削除はできません。

### メンバーの編集

各メンバー行の「編集」ボタンから以下の項目を変更できます。

| 項目 | 説明 |
|---|---|
| 表示名 | アプリ内での表示名（デフォルトは GitHub ログイン名） |
| 1日の稼働時間 | 担当者別工数グラフ等で使用（デフォルト: 6時間/日） |

GitHub ログイン名（`@username`）は変更できません。

---

## ラベル管理

**URL**: `/settings/labels`

GitHub Issue のラベルに対して、ベロシティ計算の除外設定を行います。

### ベロシティ管理フラグ

各ラベルのトグルスイッチで `include_velocity`（管理対象）を切り替えます。

- **オン（管理対象）**: 通常通りベロシティに含まれます（デフォルト）
- **オフ（除外）**: このラベルが付いた Issue はベロシティ計算から除外されます

> ラベル一覧は GitHub 同期後に表示されます。同期前は空欄になります。

### 用途例

`spike`・`tech-debt`・`管理` などの非開発 Issue を除外することで、実際の開発速度をより正確に計測できます。

## 実績種別管理

**URL**: `/settings/work-log-categories`

実績入力（ワークログ）で使用する種別（カテゴリ）とそのグループを管理します。

### グループ

カテゴリを論理的にまとめるグループです。

| 項目 | 説明 |
|---|---|
| グループ名 | 表示名（例: 「開発」「会議」） |
| 工数あり/なし | オフにすると、このグループのカテゴリで記録した実績は工数集計から除外されます |

### カテゴリ

各グループ下に属する実績の種別です。

| 項目 | 説明 |
|---|---|
| ラベル | 実績入力時に表示される名称 |
| 色 | カレンダー上の表示色 |
| グループ | 所属グループ（ドロップダウンで変更可） |
| 並び順 | 実績入力フォームでの表示順 |

カテゴリの編集はテーブル行をクリックするとインライン編集モードになります。「保存」で確定、「×」でキャンセルします。

## 祝日管理

**URL**: `/settings/holidays`

勤怠管理・実績入力カレンダーで使用する祝日・特別休日を管理します。

### 日本の祝日の自動取得

「日本の祝日を自動取得」ボタンをクリックすると、外部 API（`holidays.googl` 等）から当年の日本の祝日データを一括インポートします。既存の同じ日付のデータとは重複しません。

### 手動追加

日付と祝日名を入力して「追加」ボタンをクリックすると手動登録できます。企業独自の特別休日（創立記念日など）を追加する際に使用します。

### 削除

各行の「削除」ボタンをクリックすると削除されます。
