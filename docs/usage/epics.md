# エピック

**URL**: `/epics`

複数の Issue をまとめる大きな機能単位（エピック）を管理します。Issue とエピックを紐付けることで、機能ごとの進捗を把握できます。

---

## エピック一覧

登録済みのエピック一覧が表示されます。各エピックには以下が表示されます。

| 表示項目 | 説明 |
|---|---|
| タイトル | エピック名 |
| 説明 | エピックの概要（任意） |
| ステータスバッジ | `active` / `completed` |
| 進捗バー | `closed` Issue 数 / 全 Issue 数のパーセンテージ |
| Issue 数 | `完了数 / 総数`（例: `3 / 7`） |
| 推定工数 | 紐付く Issue のストーリーポイント合計 |

---

## エピックの作成

「+ エピックを追加」ボタンをクリックするとフォームが展開します。

| 入力項目 | 必須 | 説明 |
|---|---|---|
| タイトル | ✅ | エピックの名前 |
| 説明 | - | エピックの概要・目的 |

---

## エピックの編集

各エピックの「編集」ボタンから、タイトル・説明・ステータスを変更できます。

### ステータス

| 値 | 説明 |
|---|---|
| active | 進行中（デフォルト） |
| completed | 完了済み |

---

## エピックの削除

「削除」ボタンをクリックすると確認なしに削除されます。削除してもエピックに紐付いていた Issue が削除されることはありませんが、Issue の `epic_id` は `null` になります。

> 削除は取り消せません。ご注意ください。

---

## Issue とエピックの紐付け

Issue とエピックの紐付けは GitHub 同期では行われません。現時点では DB 直接操作 or tinker で設定できます。

```bash
php artisan tinker --execute "
App\Models\Issue::find(5)->update(['epic_id' => 1]);
"
```

> 今後のバージョンで Issue 詳細画面からエピックを選択できるようにする予定です。

---

## 進捗バーの計算

```
進捗 % = closed な Issue 数 ÷ 全 Issue 数 × 100
```

Issue にストーリーポイントが設定されていない場合でも、件数ベースで進捗が計算されます。
