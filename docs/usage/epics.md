# エピック（案件）

**URL**: `/epics`

複数の Issue をまとめる大きな機能単位（エピック／案件）を管理します。Issue とエピックを紐付けることで、機能ごとの進捗や工数をまとめて把握できます。

---

## エピック一覧

登録済みのエピック一覧が表示されます。各エピックには以下が表示されます。

| 表示項目 | 説明 |
|---|---|
| タイトル | エピック名 |
| 説明 | エピックの概要（任意） |
| ステータスバッジ | `計画中` / `進行中` / `完了` |
| 進捗バー | `closed` Issue 数 / 全 Issue 数のパーセンテージ |
| open Issue 数 | 未完了の Issue 件数 |
| 実績 / 予定 (h) | 配下の全 Task の工数サマリー（`actual_hours` 合計 / `estimated_hours` 合計）。Task が未登録または工数が未入力の場合は非表示 |
| 推定スプリント数 | 残ポイント ÷ 平均ベロシティで算出した残りスプリント数 |
| 推定工数 | 残スプリント数 × チーム日次稼働時間 × 営業日数 |

---

## 工数サマリーの仕組み

```
Epic（案件）
  └─ Story Issue（epic_id で紐付け）
       └─ Task Issue（parent_issue_id で紐付け、GitHub Sub-issues）
            └─ estimated_hours / actual_hours（スプリント詳細で入力）
```

Epic の工数サマリーは **Task Issue の工数を Story → Epic の順に積み上げた合計値**です。

- Story に Task がない場合、またはすべての Task の工数が未入力の場合は工数サマリーは表示されません
- Story に紐付く Task が1件でも工数を持つ場合に表示されます

---

## エピックの作成

「+ 新規エピック（案件）」ボタンをクリックするとフォームが展開します。

| 入力項目 | 必須 | 説明 |
|---|---|---|
| タイトル | ✅ | エピックの名前 |
| 説明 | - | エピックの概要・目的 |
| ステータス | ✅ | `計画中` / `進行中` / `完了` |

---

## エピックの編集

各エピックの「編集」ボタンから、タイトル・説明・ステータスを変更できます。

### ステータス

| 値 | 説明 |
|---|---|
| 計画中 | 計画段階（デフォルト） |
| 進行中 | 開発中 |
| 完了 | 完了済み |

---

## エピックの削除

「削除」ボタンをクリックすると確認なしに削除されます。削除してもエピックに紐付いていた Issue が削除されることはありませんが、Issue の `epic_id` は `null` になります。

> 削除は取り消せません。ご注意ください。

---

## Issue とエピックの紐付け

Issue とエピックの紐付けは GitHub 同期では行われません。**スプリント詳細画面の Issue 一覧**から UI で設定できます。

### スプリント詳細からの紐付け手順

1. `/sprints/{id}` を開き「Issue 一覧」タブを表示する
2. 紐付けたい Issue 行のエピック選択ドロップダウンをクリックする
3. 一覧からエピック（案件）を選択すると即座に保存される
4. 紐付けを解除したい場合は「案件なし」を選択する

---

## 進捗バーの計算

```
進捗 % = closed な Issue 数 ÷ 全 Issue 数 × 100
```

Issue にストーリーポイントが設定されていない場合でも、件数ベースで進捗が計算されます。
