# スプリント

## スプリント一覧

**URL**: `/sprints`

スプリントの一覧です。

### 表示内容

各スプリントカードには以下が表示されます。

- スプリント名
- 期間（開始日 〜 終了日）・稼働日数
- 状態バッジ（`open` / `closed`）
- ポイント速度・Issue 速度
- 完了 Issue 数 / 総 Issue 数

### スプリントの作成

スプリント作成方法はリポジトリの同期モードによって異なります。

#### Iteration モード（`github_project_number` 設定済み）

GitHub Projects の `Sprint` Iteration フィールドから自動的に同期されます。
GitHub 側で Sprint Iteration を作成後、「GitHub 同期」ボタンを押すとスプリントが作成されます。

#### 同期なしモード（`github_project_number` 未設定）

GitHub との連携なしの場合は、スプリントの自動作成は行われません。
`github_project_number` を設定して Iteration モードに移行するか、DB または tinker で直接登録してください。

## スプリント詳細

**URL**: `/sprints/{id}`

スプリントの詳細を3つのタブで確認できます。

---

### タブ 1: Issue 一覧

スプリントに紐付いた Issue（Story）の一覧です。各 Story の下にサブイシュー（Task）がインデントして表示されます。

#### Story（親 Issue）の表示項目

| 表示項目 | 説明 |
|---|---|
| 状態ドット | 緑 = open、グレー = closed |
| Issue 番号 | GitHub の Issue 番号（`#123` 形式） |
| タイトル | Issue のタイトル |
| エピック選択 | エピック（案件）を選択するドロップダウン（紫）。変更すると即座に保存される |
| ラベル | Issue に付いているラベル |
| 担当者 | アサインされたユーザー名（`@username`） |
| ポイント | ストーリーポイント（青バッジ） |

#### Task（Sub-issue）の表示項目

GitHub Sub-issues として登録されたタスクは、親 Story の下にインデント表示されます。

| 表示項目 | 説明 |
|---|---|
| 状態ドット | 緑 = open、グレー = closed |
| タイトル | タスクのタイトル |
| 担当者 | アサインされたユーザー名 |
| 予定 (h) | 予定工数の入力フィールド（フォーカスを外すと自動保存） |
| 実績 (h) | 実績工数の入力フィールド（フォーカスを外すと自動保存） |

#### 工数の入力

予定工数・実績工数は Task 行の入力フィールドに直接入力できます。フィールドからフォーカスが外れると自動的に保存されます（`PATCH /issues/{id}`）。

- 入力できる値: `0` 以上 `9999.99` 以下の数値
- 未入力にする場合はフィールドを空にして保存

---

### タブ 2: バーンダウンチャート

スプリント期間のポイント消化状況を折れ線グラフで表示します。

- **理想線**: 総ポイントを期間で均等割りした理想的な消化ライン
- **実績線**: 実際の Issue クローズ日時に基づく残ポイントの推移

> チャートは `story_points` が設定されている Issue のみを対象にします。ポイント未設定の Issue は 0 として扱われます。

---

### タブ 3: 担当者別

スプリント内のオープン Issue を担当者別に集計した棒グラフです。

- **open Issue 数**: 担当者ごとの未完了 Issue 件数
- **合計ポイント**: 担当者ごとのオープン Issue のストーリーポイント合計

チームの負荷バランスを確認するために使います。

---

## ヘッダーの速度指標

スプリント詳細ページのヘッダーには以下が表示されます。

| 指標 | 計算方法 |
|---|---|
| ポイント速度 | クローズ済み・ベロシティ除外なし Issue の `story_points` 合計 |
| Issue 速度 | クローズ済み・ベロシティ除外なし Issue の件数 |
| 完了 Issue | `closed` の Issue 数 / 全 Issue 数 |

> `exclude_velocity = true` の Issue、または `exclude_velocity = true` のラベルを持つ Issue は速度計算から除外されます。

## スプリントの開始日・稼働日数の設定

開始日と稼働日数は GitHub には存在しないため、アプリ側で手動設定が必要です（現在は DB 直接設定 or tinker で対応）。

```bash
php artisan tinker --execute "
App\Models\Sprint::find(1)->update([
    'start_date' => '2025-04-01',
    'working_days' => 10,
]);
"
```

> UI から設定する機能は未実装です。
