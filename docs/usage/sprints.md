# スプリント

## スプリント一覧

**URL**: `/sprints`

スプリントの一覧です。

### 表示内容

各スプリントカードには以下が表示されます。

- スプリント名
- 期間（開始日 〜 終了日）・稼働日数
- スプリントゴール（設定されている場合）
- 状態バッジ（`open` / `closed`）
- ポイント速度・Issue 速度
- 完了 Issue 数 / 総 Issue 数

### スプリントの作成

スプリント作成方法はリポジトリの同期モードによって異なります。

#### Iteration モード（`github_project_number` 設定済み）

GitHub Projects の `Sprint` Iteration フィールドから自動的に同期されます。
GitHub 側で Sprint Iteration を作成後、「GitHub 同期」ボタンを押すとスプリントが作成されます。

#### 同期なしモード（`github_project_number` 未設定）

GitHub との連携なしの場合は、スプリントの自動作成は行われません。
`github_project_number` を設定して Iteration モードに移行するか、DB または tinker で直接登録してください。

## スプリント計画

**URL**: `/sprints/{id}/plan`

スプリントのキャパシティ計画と Issue 管理を行う画面です。

### スプリントゴール

ページ上部でスプリントゴールを設定・編集できます。「保存」ボタンで即座に更新されます。

### キャパシティ計画

メンバー別の稼働キャパシティとアサイン状況を確認できます。

| 列 | 説明 |
|---|---|
| メンバー | GitHub ログイン名 / 表示名 |
| 日次時間 | メンバー設定の `daily_hours` |
| キャパシティ (h) | 日次時間 × スプリント稼働日数 |
| アサイン PT | 担当 Issue のストーリーポイント合計 |
| アサイン件数 | 担当 Issue 件数 |

### Issue 管理

スプリントに割り当て済みの Issue 一覧とバックログの Issue 一覧を並べて表示します。
「バックログに戻す」「スプリントに追加」ボタンでスプリントへの割り当てを変更できます。

### Definition of Ready (DoR) チェックリスト

折り畳み可能な DoR チェックリストが表示されます。スプリントに着手する前に Issue が「着手可能な状態か」を確認するために使います。
全項目にチェックが入ると「✅ 全項目クリア！着手可能な状態です」と表示されます。

> チェック項目は設定画面（`/settings/general`）でカスタマイズできます。チェック状態はブラウザのセッション内のみ保持され、サーバーには保存されません。

### Definition of Done チェックリスト

折り畳み可能な DoD チェックリストが表示されます。スプリント計画時に完了条件を確認するために使います。
全項目にチェックが入ると「✅ 全項目クリア！」と表示されます。

> チェック状態はブラウザのセッション内のみ保持され、サーバーには保存されません。

### プランニングポーカー

スプリントに割り当てられた Issue に対して、チームでフィボナッチ数列のポイントを投票する機能です。

- 「プランニングポーカー」セクションを展開すると Issue 一覧が表示されます
- Issue をクリックすると投票パネルが開きます
- **ポイント選択**: 1 / 2 / 3 / 5 / 8 / 13 / 21 のボタンから選択
- **結果表示**: 各メンバーの投票、平均、中央値、合意ポイントが表示されます
  - 全員が同じ値を投票した場合: 🟢「合意: N pt」バッジ
  - 意見が分かれた場合: 🟡「未合意」バッジ
- 「取り消す」ボタンで自分の投票を削除できます

> 投票はサーバーに保存されます。ポーカーセッション後にスプリント計画を続行できます。

## スプリントボード

**URL**: `/sprints/{id}/board`

GitHub Projects のステータスに基づく 4 列のカンバンボードです。

### 列構成

| 列 | 対応ステータス |
|---|---|
| Todo | Todo |
| In Progress | In Progress |
| In Review | In Review |
| Done | Done |

Issue の「→」ボタンで次のステータスに進めます（`PATCH /issues/{id}`）。

### IssueCard の表示項目

- Issue 番号・タイトル
- エピック・ラベル
- 担当者・ストーリーポイント
- 🚫 ブロッカーバッジ（`is_blocker = true` の場合、理由も表示）
- PR バッジ（紐付いている PR の番号・タイトル・レビュー状態・CI ステータス）

### PR バッジの見方

| バッジ | 意味 |
|---|---|
| `Merged` | PR がマージ済み |
| `Closed` | PR がクローズ済み（未マージ） |
| `Approved` | レビュー承認済み |
| `Changes` | 変更リクエストあり |
| `Open` | レビュー待ち |
| ✅ | CI 成功 |
| ❌ | CI 失敗 |
| ⏳ | CI 実行中 |

### WIP 制限

列ごとに WIP（Work In Progress）上限を設定できます。設定は `設定 > 一般` の「WIP 制限」セクションで行います。

- 上限を超えた列のヘッダーは赤くハイライトされ「WIP超過」バッジが表示されます
- カウントは `現在のIssue数 / 上限` の形式で表示されます
- 上限 0 = 無制限（デフォルト）

### GitHub Projects 書き戻し（双方向同期）

ボード上でステータスを変更すると、GitHub Projects の Status フィールドにも自動的に書き戻されます（GraphQL Mutation 経由）。

> `github_project_number` が設定されているリポジトリのみ有効です。

### PR 同期

ヘッダーの「PR 同期」ドロップダウンからリポジトリを選択すると、そのリポジトリの PR を同期できます（`POST /repositories/{id}/sync-prs`）。

## スプリント詳細

**URL**: `/sprints/{id}`

スプリントの詳細を 3 つのタブで確認できます。

### ヘッダー

- スプリントゴール（設定されている場合）
- 「Issue のクローズ / オープン切替」ボタン

### タブ 1: Issue 一覧

スプリントに紐付いた Issue（Story）の一覧です。各 Story の下にサブイシュー（Task）がインデントして表示されます。

#### Story（親 Issue）の表示項目

| 表示項目 | 説明 |
|---|---|
| 状態ドット | 緑 = open、グレー = closed |
| Issue 番号 | GitHub の Issue 番号（`#123` 形式） |
| タイトル | Issue のタイトル |
| エピック選択 | エピック（案件）を選択するドロップダウン（紫）。変更すると即座に保存される |
| ラベル | Issue に付いているラベル |
| 担当者 | アサインされたユーザー名（`@username`） |
| ポイント | ストーリーポイント（青バッジ） |
| ブロッカー | 🚫 バッジ（is_blocker = true の場合） |
| クローズ / オープン | GitHub 状態を切り替えるボタン |
| ゴール紐付け | 「ゴール紐付け」ボタンでスプリントゴールに Issue を関連付ける。紐付け済みは 🎯「ゴール」バッジで表示（クリックで解除） |

#### Task（Sub-issue）の表示項目

GitHub Sub-issues として登録されたタスクは、親 Story の下にインデント表示されます。

| 表示項目 | 説明 |
|---|---|
| 状態ドット | 緑 = open、グレー = closed |
| タイトル | タスクのタイトル |
| GitHub Projects 日程 | プロジェクトの開始日・完了目標日（設定されている場合） |
| 担当者 | アサインされたユーザー名（`@username`） |
| 予定 (h) | 予定工数の入力フィールド（フォーカスを外すと自動保存） |
| 実績 (h) | ワークログの合計工数（読み取り専用）。消化率バッジ（%）も表示される |
| 記録リンク | 「記録」をクリックするとワークログ入力画面に遷移 |
| GitHub リンク | GitHub Issue ページへの外部リンク |

#### 工数の入力

予定工数は Task 行の入力フィールドに直接入力できます。フィールドからフォーカスが外れると自動的に保存されます（`PATCH /issues/{id}`）。

### タブ 2: バーンダウンチャート

スプリント期間のポイント消化状況を折れ線グラフで表示します。

- **理想線**: 総ポイントを期間で均等割りした理想的な消化ライン
- **実績線**: 実際の Issue クローズ日時に基づく残ポイントの推移

> チャートは `story_points` が設定されている Issue のみを対象にします。ポイント未設定の Issue は 0 として扱われます。

### タブ 3: 担当者別

スプリント内のオープン Issue を担当者別に集計した棒グラフです。

## 累積フロー図 (CFD)

**URL**: `/sprints/{id}/cfd`

スプリント期間の Issue 完了 / 未完了の推移を積み上げエリアチャートで表示します。

- **完了（Done）**: クローズ済み Issue 数の累積推移（緑）
- **未完了（Open）**: オープン Issue 数の推移（青）

スプリント開始日から終了日まで 1 日単位でプロットされます。
未来の日付は表示されません。

## スプリントレビュー

**URL**: `/sprints/{id}/review`

スプリントレビューセレモニーの記録を残す画面です。

### 記録できる項目

| 種別 | 説明 |
|---|---|
| デモ | デモした機能・内容の記録 |
| フィードバック | ステークホルダーからのフィードバック |
| 受入判断 | Issue の受け入れ判断（受入 / 持越） |

### 受入判断の記録

Issue を選択し、結果（`受入` または `持越`）を記録できます。受入判断は特定の Issue に紐付けて保存されます。

### 記録の一覧

登録済みのレビュー記録が種別・結果バッジ付きで一覧表示されます。削除も可能です。

### CSV ダウンロード

ページ右上の「CSV ダウンロード」ボタンをクリックすると、スプリントの完了 Issue 一覧を CSV 形式でダウンロードできます。

出力内容: Issue 番号・タイトル・担当者・ストーリーポイント・状態・エピック・ラベル

> Excel での文字化けを防ぐため UTF-8 BOM 付きで出力されます。

## ブロッカートラッキング

Issue の `is_blocker` フラグと `blocker_reason` フィールドでブロッカーを管理します。

- スプリントボード・デイリースクラム画面で 🚫 バッジとして表示されます
- GitHub 同期では上書きされません（アプリ内で手動管理）

## スプリント完了フロー

スプリント一覧（`/sprints`）から各スプリントの「完了」ボタンをクリックすると、スプリント完了ウィザードが起動します。

### 完了ウィザードの手順

1. **未完了 Issue の確認**: 残っているオープン Issue の一覧が表示されます
2. **各 Issue の処置を選択**:
   | 選択肢 | 動作 |
   |---|---|
   | 次スプリントへ持ち越す | 選択した次スプリントに Issue を移動し、持ち越し理由を記録できます |
   | バックログに戻す | Issue のスプリント割り当てを解除してバックログに戻します |
   | このスプリントに残す | 変更なし（スプリントはクローズされますが Issue は残ります） |
3. **「スプリントを完了する」**: 確定するとスプリントの状態が `closed` に変わります

> スプリントがすでに `closed` の場合は完了ボタンが非表示になります。

## スプリント持ち越し

未完了 Issue をバックログに戻す操作は、スプリント完了ウィザードから行えます。持ち越し理由は `carry_over_reason` として Issue に記録されます。

## ヘッダーの速度指標

スプリント詳細ページのヘッダーには以下が表示されます。

| 指標 | 計算方法 |
|---|---|
| ポイント速度 | クローズ済み・ベロシティ除外なし Issue の `story_points` 合計 |
| Issue 速度 | クローズ済み・ベロシティ除外なし Issue の件数 |
| 完了 Issue | `closed` の Issue 数 / 全 Issue 数 |

> `exclude_velocity = true` の Issue、または `exclude_velocity = true` のラベルを持つ Issue は速度計算から除外されます。

## スプリントの開始日・稼働日数の設定

開始日と稼働日数は GitHub には存在しないため、アプリ側で手動設定が必要です（現在は DB 直接設定 or tinker で対応）。

```bash
php artisan tinker --execute "
App\Models\Sprint::find(1)->update([
    'start_date' => '2025-04-01',
    'working_days' => 10,
]);
"
```

> UI から設定する機能は未実装です。
