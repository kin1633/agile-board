# バックログ

**URL**: `/backlog`

スプリントに割り当てられていない Issue（バックログ）を管理する画面です。

---

## 表示内容

### Issue 一覧

スプリント未割当の Issue が一覧表示されます。各行に以下が表示されます。

| 表示項目 | 説明 |
|---|---|
| Issue 番号 | GitHub の Issue 番号（`#123` 形式） |
| タイトル | Issue のタイトル |
| エピック | 紐付いているエピック（案件）名 |
| ラベル | Issue に付いているラベル |
| 担当者 | アサインされたユーザー名 |
| ポイント | ストーリーポイント（青バッジ） |
| スプリット割り当て | ドロップダウンからスプリントを選んで即座に割り当て |

---

## フィルタリング

| フィルター | 説明 |
|---|---|
| エピック | 特定のエピックに絞り込む |
| 担当者 | 特定の担当者に絞り込む |

フィルターを変更するとページが自動的に再読み込みされます。

---

## スプリントへの割り当て

Issue 行のスプリット選択ドロップダウンからアクティブなスプリントを選択すると、即座に割り当てられます（`PATCH /backlog/{issue}/assign-to-sprint`）。

割り当てると Issue はバックログ一覧から消えます。

---

## Issue の作成

「+ 新規 Issue 作成」ボタンをクリックするとフォームが展開します。

| 入力項目 | 必須 | 説明 |
|---|---|---|
| リポジトリ | ✅ | 作成先のリポジトリを選択 |
| タイトル | ✅ | Issue のタイトル |
| 説明 | — | Issue の詳細説明（Markdown 対応） |

「作成」ボタンをクリックすると GitHub に Issue が作成され、即座にバックログ一覧に追加されます（`POST /backlog/issues`）。

---

## ストーリーポイントのインライン編集

Issue 行のポイントバッジ（または「ポイント設定」テキスト）をクリックすると入力フィールドが展開します。

- 数値を入力して `Enter` キーまたはフィールドのフォーカスアウトで保存されます（`PATCH /issues/{id}/fields`）
- `Escape` キーで編集をキャンセルします

---

## バックログへの戻し方

スプリント計画画面（`/sprints/{id}/plan`）またはスプリント完了ウィザードから Issue をバックログに戻せます。
