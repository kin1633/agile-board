# デイリースクラム

**URL**: `/daily-scrum`

デイリースクラムセレモニーの進捗記録を管理する画面です。
タスク（Sub-issue）起点で進捗 % とメモを記録します。

---

## 画面構成

### フィルター

| フィルター | 説明 |
|---|---|
| スプリント | 記録対象のスプリントを選択（アクティブスプリントがデフォルト） |
| 日付 | 記録日（デフォルト: 今日） |
| メンバー | 記録するメンバーを選択（ログインユーザーがデフォルト） |

### 記録フォーム

| 入力項目 | 説明 |
|---|---|
| タスク | 対象タスク（Sub-issue）を選択。選択肢には「ストーリー名 / エピック名」が付記される |
| 進捗 | 進捗率（0 〜 100%）を入力 |
| メモ | 任意メモ（ブロッカー・状況メモなど） |

「記録」ボタンで保存します（`POST /daily-scrum`）。

---

## 記録一覧

当日・選択メンバーの記録一覧が表示されます。

| 表示項目 | 説明 |
|---|---|
| タスク名 | タスクのタイトル（Issue 番号付き） |
| ストーリー名 | 親 Issue のタイトル |
| エピック名 | 紐付いているエピック名 |
| 進捗 % | 記録した進捗パーセンテージ |
| メモ | 記録したメモ |
| 削除 | 記録を削除するボタン |

---

## タスク選択肢の絞り込み

タスクの選択肢には、選択中のスプリントに含まれる**オープンなタスク（Sub-issue）**のみが表示されます。
クローズ済みのタスクは選択肢から除外されます。

---

## 運用上のポイント

- 毎朝デイリースクラム開始前に各メンバーが前日の進捗を入力しておくと、スプリントの進捗把握に役立ちます
- ブロッカーがある場合はメモ欄に記録し、スプリントボードでのブロッカーフラグと合わせて管理します
